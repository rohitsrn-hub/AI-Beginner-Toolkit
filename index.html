Step 1 · Prompt engine

Lyra — your prompt engineer before any AI.

Goal: turn vague ideas into sharp, token‑efficient prompts.
What Lyra is

A 4‑step prompt optimisation layer.

Lyra is a custom prompt optimiser you run before Claude, Gemini, or any other tool. It applies the 4‑D method to your rough idea so your final prompt is clear, compact, and much cheaper to run.

4‑D method
Deconstruct: pull out intent, entities, context, audience, constraints.
Diagnose: spot ambiguity, missing detail, unnecessary fluff.
Develop: choose pattern — creative, technical, educational, complex.
Deliver: output a structured, ready‑to‑use prompt plus usage notes.
Operating modes
DETAIL mode: “DETAIL using Claude — [rough idea]”. Lyra asks 2‑3 questions, then builds the prompt.
BASIC mode: “BASIC using Claude — [rough idea]”. Quick clean‑up and compression.
Every answer ends with what changed, techniques used, and a pro tip.
Daily usage examples

How to run ideas through Lyra.

Draft email → BASIC using Claude — write an email to my manager asking for leave next Monday and Tuesday.
Build app → DETAIL using Claude — build a stock entry web app for my canteen staff with barcode search and low‑stock alerts.
Research → BASIC using Claude — compare demand forecasting methods for 5000 SKUs in a retail store.
Analyse data → DETAIL using Claude — analyse my sales Excel and find weekday vs weekend patterns.
Step 2 · First real project

FinSight — a personal finance tracker built entirely by AI.

Goal: go from prompt to working HTML app you can deploy.
What FinSight is

Single‑file finance dashboard from your bank PDFs.

FinSight is a mobile‑friendly web app that reads your bank statement PDFs in the browser, categorises expenses, shows month‑over‑month trends, and suggests saving ideas. Everything runs client‑side, so nothing gets uploaded to a server.

Client‑side PDF parsing via PDF.js — supports SBI / HDFC / ICICI / Axis formats.
Auto‑categorisation into Groceries, Food & Dining, Transport, Utilities, Shopping, Entertainment, Health, EMI/Finance, Others.
Charts with Chart.js: category breakdown, month comparison, top expense categories.
AI‑style saving recommendations based on category shares and spikes.
Budget planner with per‑category limits and green / amber / red progress bars.
“Load sample data” mode so you can demo the app without using real statements.
How to build FinSight with Claude

From prompt to finsight.html.

Ask Lyra in DETAIL mode to refine the FinSight build prompt for your banks and categories.
Paste the full prompt into Claude and say: “Generate a complete, single HTML file called finsight.html with inline CSS and JS.”
Save the response as finsight.html, open it in Chrome, and test using the sample data button.
If parsing fails for your bank, tell Claude: “My bank is HDFC, here are the column headers — fix the parser regex.”
Deployment idea

Host FinSight for free with GitHub Pages.

Create a new GitHub repo, add finsight.html at the root, and commit.
In Settings → Pages, select the main branch and root folder. GitHub serves it at a public URL.
Share this URL with family or colleagues as “FinSight — built with Claude”.
Step 3 · Market workflow

KIRAN — a Gemini Gem for Indian stock market briefs.

Goal: replace random stock tips with structured daily education.
What this Gem does

A disciplined morning market advisor, not a tipster.

KIRAN (Key Investment Recommendation and Analysis Node) is a Gemini Gem that runs a consistent morning briefing for Indian retail investors, using live market data and a fixed, educational format.

MORNING MARKET BRIEF: NIFTY 50 signal, FII/DII flows, global cues, USD/INR, and key events for the day.
SECTOR WATCH: 2–3 sectors highlighted with reasons and example stocks to watch.
STOCKS TO WATCH: 3–5 names with price, why watching, key level, and risk notes.
AVOID TODAY: 2–3 stocks/sectors with short‑term headwinds.
LEARNING MOMENT: one technical/fundamental concept explained in two sentences.
How to create and use KIRAN

Turn it into your fixed morning ritual.

In Gemini, click Gems → New Gem, paste the KIRAN system prompt into Instructions, and name it “Morning Market Advisor”.
Every weekday at ~7:30 AM, open the Gem and type: “Good morning KIRAN, what’s today looking like?”
Read the brief in 3–5 minutes, note key levels, and compare with live action once the market opens.
In the evening, ask: “KIRAN, how did the morning watchlist do today?” to close the loop.
Risk & compliance

Strictly education, not SEBI‑registered advice.

KIRAN always reminds you: this is educational only, not investment advice.
Risk profiles (Conservative / Moderate / Aggressive) guide how it talks about instruments.
If you ask “Should I buy X?”, it explains how to analyse X instead of saying yes/no.
Step 4 · Capture everything

Claude as a voice‑to‑task manager for your brain.

Goal: turn messy voice dumps into an actionable to‑do system.
Workflow overview

From “rambling into phone” to prioritised action list.

Instead of buying yet another productivity app, you can use your phone’s built‑in voice input plus Claude to convert raw thoughts into a structured task list with priorities, time estimates, and owners.

iPhone: Shortcuts → Dictate Text → add organiser prompt → copy to clipboard → open Claude.
Android: Tasker → record voice → send transcript + prompt to Claude → push to Google Tasks.
Any device: open Claude app, tap mic, dump everything, then paste the organiser prompt below your transcript.
Voice task organiser prompt

Sections, verbs, estimates, and a daily total.

The prompt tells Claude to organise your voice dump into sections like Urgent + Important, Important, Nice to Do, Waiting on Others, Ideas to Explore, and Ignore.

Each task starts with a clear verb: Call / Email / Write / Fix / Review / Book / Buy / Ask.
Claude adds estimated time (5 min / 30 min / 2 hrs / half day) and any deadlines you implied.
When you mention people, tasks get tagged with @Name for easy filtering.
It finishes with TOTAL TASKS: X | Estimated time today: Y hrs and asks if you want to reprioritise.
Step 5 · Learn with AI

NotebookLM — a private AI study partner.

Goal: build a topic‑specific second brain, starting with “How to use AI”.
What NotebookLM does

Trains on your sources only, with citations.

NotebookLM lets you upload PDFs, web pages, YouTube videos, Google Docs, and audio. It then creates a private notebook where every answer is grounded in those sources and fully cited, so you can trust where information comes from.

Add videos like Matt Wolfe and Andrej Karpathy, Claude usage guides, prompting tutorials, and your own project docs.
Use Audio Overview to generate a podcast‑style summary of your sources for commuting.
Use Study Guide, Briefing Doc, and Mind Map to turn raw content into structured learning materials.
Ask questions like “What did these sources NOT cover?” to discover gaps in your understanding.
Pro research loop

Grow a deep AI notebook over time.

Pick a topic (e.g. AI agents), add 3–5 good articles or videos to NotebookLM.
Generate an Audio Overview and listen once.
Ask: “What are the main frameworks and trade‑offs mentioned here?” and “What did these sources miss?”
Add the missing sources NotebookLM suggests, then repeat every week. The notebook becomes your curated knowledge base.

Step 7 · Save & ship

GitHub, Pages, Vercel — from prompt to live app.

Goal: never lose AI‑generated code, and deploy it on the web for free.
What a repo is

Your permanent home for AI‑written code.

A GitHub repository is a cloud folder with history. It stores your code, documentation, and issues, and connects directly to hosting platforms like GitHub Pages, Vercel, and Render.

Use it to save full apps that Claude generates, like finsight.html.
Track improvements over time as you refine prompts and architecture.
Collaborate with others — they can open issues, fork, and send pull requests.
Deploying for free

Static sites, full‑stack apps, and databases.

GitHub Pages · static sites
Best for single‑file or front‑end projects (like FinSight).
Put index.html at the repo root, enable Pages, and get a public URL.
Vercel / Render · dynamic apps
Connect your GitHub account, choose the repo, accept auto‑detected settings.
Perfect for Next.js, Node, or Python projects that Claude scaffolds for you.
MongoDB Atlas · storage
Create a free cluster and copy the connection string.
Add MONGODB_URI as an environment variable in Vercel or Render.
Tell Claude: “Wire this app to MongoDB Atlas using process.env.MONGODB_URI and Mongoose.”
